# Window Groups — Менеджер групп окон для macOS

## Цель

Приложение для macOS, которое запоминает группы окон приложений, их размеры, расположение и восстанавливает всё одной кнопкой/хоткеем.

## Проблема пользователя

Пользователи macOS тратят время на ручное размещение окон при многозадачности и работе на 2+ дисплеях.

## Целевая аудитория

1. Разработчики и инженеры (IDE + браузер + терминал + docs)
2. PM/дизайнеры (Notion/Figma/Slack/браузер)
3. Пользователи с несколькими мониторами
4. Power users, использующие хоткеи

---

## Scope

### MVP

1. **Window Capture**
   - Запоминание позиций, размеров и мониторов окон
   - Захват всех окон текущего workspace

2. **Group Management**
   - Сохранение выбранных пользователем окон в группу
   - Авто-именование группами городов (small towns → megacities)
   - Удаление групп

3. **Group Restore**
   - Восстановление группы по хоткею или из меню
   - Запуск неактивных приложений с последним открытым файлом

4. **Menu Bar UI**
   - NSStatusItem с выпадающим меню
   - Список сохранённых групп
   - Пункт «Создать группу» → режим выбора окон
   - Пункт «Сохранить группу» (после выбора окон)
   - Восстановление группы из меню
   - Настройки приложения

5. **Multi-monitor Support**
   - Работа с 1..N дисплеями
   - Корректное позиционирование при разных scaling

6. **Global Shortcuts**
   - Назначение хоткеев на восстановление групп
   - Move/resize окна под курсором

### V2 (после MVP)

- iCloud sync настроек
- Per-app rules (авторасположение для конкретных приложений)
- Export/Import групп
- Ручное переименование групп

### Out of Scope (MVP)

- AI-функции автопланировки
- Совместная работа в реальном времени
- App Store (direct distribution)

## Функциональные требования

### FR-1 Window Capture
- Захват всех окон текущего workspace
- Сохранение: позиция (x, y), размер (width, height), монитор, app bundle ID, окно maximized/minimized

### FR-2 Group CRUD

**Create — процесс создания группы:**
1. Пользователь нажимает «Создать группу» в меню
2. Приложение переходит в режим выбора окон
3. Пользователь кликает по заголовкам нужных окон
4. При клике на окно: визуальная подсветка (свечение по всей рамке окна), окно добавляется в группу
5. Пункт меню меняется на «Сохранить группу»
6. Пользователь нажимает «Сохранить группу»
7. Приложение сохраняет выбранные окна как группу
8. Подсветка окон гасится, работа с ОС продолжается
9. В меню появляется новый пункт с названием группы (город)

**Read:** отображение списка групп в меню

**Delete:** удаление группы из меню

### FR-3 Group Restore
- Применение сохранённой группы к текущему workspace
- Если приложение не запущено — запустить его
- Если приложение запущено без окон — открыть последний документ (MRU)
- Если окно другого размера — ресайз к сохранённому

### FR-4 Multi-monitor
- Определение подключённых дисплеев
- Корректировка позиций при восстановлении на другой конфигурации мониторов

### FR-5 Shortcuts
- Глобальные хоткеи для быстрого восстановления групп
- Системные горячие клавиши macOS

### FR-6 Export/Import (V2)
- Экспорт/импорт групп в JSON

## Нефункциональные требования

- **Performance:** действие restore ≤ 500ms
- **Stability:** crash-free sessions ≥ 99%
- **Resource usage:** idle CPU < 2%, RAM < 100 MB
- **Privacy:** локальная обработка; без отправки данных
- **Security:** требуется Accessibility permission

---

## UX Требования

- Menu bar app (NSStatusItem)
- Все действия доступны из меню
- Визуальный стиль: минималистичный, macOS-native
- Поддержка macOS 12+

---

## Технологии

- **Язык:** Go
- **UI:** AppKit (CGO) или Cocoa
- **Window API:** Accessibility API (AXUIElement) через CGO
- **Меню:** NSStatusItem
- **Хоткеи:** Hotkey (Go package) или MASShortcut
- **Хранение:** JSON-файлы + UserDefaults
- **Сборка:** GitHub Actions (darwin/amd64, darwin/arm64)
